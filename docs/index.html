What is TUIF?

TUIF stands for Text User Interface Framework

It can be used to make text based user interfaces that can work with different outputs, these outputs will include:

  • ANSI terminal
  • HTML
  • SDL

The first implementations will be for ANSI terminal that should work both locally and from telnet access

It has an extensible, modular system, consisting of the following modules:

  • Data store

    It holds the elements data, It can be persistent or in memory

  • Element Manager

    It manages the data, adds, removes elements

  • Layout Engine

    It handles the layout of the elements

  • Output module

    It turns the data, into actual displayable character/HTML elements/pixels

  • Input module

    It turns inputs (keyboard, touchscreen, mouse) into usable events

What is the point if this?

There are multiple reasons for this.

There is no tool that allows you to write once and use it on HTML, terminal and Native GUI app.
While HTML(+css+js) is nice and can work on even in a natively running software (and on terminal, to some degree), But it's overly complex and slow for a lot of applications..
